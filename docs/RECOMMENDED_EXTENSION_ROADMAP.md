# 推荐扩展路线

## 当前最推荐先做

1. Java 任务生命周期管理增强。
2. Java 配置中心增强。
3. Vue 接入更多平台配置。
4. Python Agent 注册中心增强。
5. SSE / Replay 过滤与回放体验优化。

这些方向能增强平台能力，同时不破坏当前 v2 演示链路。

## 当前不建议马上做

1. 用户系统。
2. 权限系统。
3. 团队协作。
4. 完整工作流动态编排替换 LangGraph。
5. 本地模型完整接入。
6. C++ Runner 完全替代 Python Runner。

原因：当前阶段优先稳定单人演示闭环，不应过早引入复杂平台治理。
