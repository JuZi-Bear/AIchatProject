# Web UI 交互规格

## 完整操作路径

用户从打开页面到运行完成的推荐路径：

1. 打开 Web UI。
2. 在左侧选择演示案例。
3. 选择模型。
4. 设置最大修复次数。
5. 选择演示模式或开发模式。
6. 开启或关闭插件。
7. 勾选人工审批确认。
8. 点击“开始运行”。
9. 查看工作流节点状态变化。
10. 查看自动修复高光或一次成功提示。
11. 查看最终结果、pytest、质量评分和插件摘要。
12. 展开报告或历史记录复盘。

## 运行前确认

运行前必须明确提示：

- 当前将运行 AI 生成的 Python 代码。
- 用户需要勾选“我确认允许运行 AI 生成的代码”。
- 未勾选时点击开始运行，只显示 warning，不执行 Runner。

建议文案：

```text
请确认允许运行 AI 生成的代码。未确认时系统只会等待审批，不会执行 Runner。
```

## 运行中状态变化

运行中页面应持续反馈：

- 当前执行 Agent。
- 工作流节点状态。
- 已完成节点。
- 是否进入自动修复。
- 当前修复轮次。

状态变化建议：

```text
Waiting   → 等待中
Running   → 运行中
Done      → 已完成
Failed    → 失败
Repairing → 修复中
```

每次 LangGraph 回调时只更新对应节点和摘要，不刷新无关区域。

## 运行后结果查看

运行完成后，用户应先看到：

- 成功 / 失败。
- run_id。
- 修复次数。
- pytest 是否通过。
- 覆盖率。
- 质量评分。
- 安全状态。
- 报告路径。
- 报告文件名。

再往下查看：

- 结果索引。
- Agent 输出 Tabs。
- stdout / stderr 折叠区。
- 插件结果。
- 完整报告。
- 历史记录。

第六步后，运行完成页的推荐查看顺序为：

```text
最终结果总览 → 结果索引 → Agent 输出详情 Tabs → 历史记录
```

`render_result_overview(ui_view_model)` 负责一屏展示最终状态；`render_result_index(ui_view_model)` 负责提供快速入口。

## 错误与修复结果如何展示

当发生错误时：

- Runner 节点显示 Failed。
- Sentry Agent 节点显示 Repairing 或 Done。
- 自动修复高光卡片显示错误摘要。
- Coder Agent 修复后显示“已生成修复版代码”。
- 如果再次运行成功，最终结果卡片显示成功。

演示模式只展示：

- 错误摘要。
- Sentry 分析摘要。
- Coder 修复摘要。
- 再次运行结果。

开发模式展示：

- 完整 stderr。
- 完整 pytest stdout / stderr。
- 完整 sentry_result。
- 完整 state。

## 如何减少查找结果的步骤

- 顶部状态摘要直接显示 success、retry_count、quality_score。
- 工作流下方直接放“最终结果卡片”。
- 工作流下方直接放“结果索引”，用于快速进入代码、测试、错误、修复、插件、报告和历史入口。
- stdout 和 stderr 使用固定标题 expander，不分散在多个区域。
- pytest 结果在结果索引中有独立 Tab，不只藏在 Tester Agent Tab 中。
- 插件结果摘要先展示，详细内容再展开。
- 报告在结果索引和 Agent 输出 Tabs 中都有入口，完整 Markdown 默认折叠。
- 历史记录默认折叠，但入口固定。

## 运行完成后如何找到关键结果

### 找到最终代码

1. 运行完成后查看“结果索引”。
2. 点击“最终代码”Tab。
3. 展开“查看最终代码”。

### 查看 pytest 测试结果

1. 运行完成后查看“结果索引”。
2. 点击“pytest 测试”Tab。
3. 查看 test_success、coverage、quality 三个指标；需要详细日志时展开 pytest stdout / stderr。

### 查看错误摘要

1. 运行完成后查看“结果索引”。
2. 点击“错误摘要”Tab。
3. 先看摘要；开发模式下可展开完整错误日志。

### 查看自动修复过程

1. 运行完成后查看“AI 工作流时间轴”的“自动修复高光时刻”。
2. 或点击“结果索引”中的“自动修复”Tab。
3. 展开 Sentry Agent 分析结果查看修复依据。

### 查看报告

1. 运行完成后查看“结果索引”。
2. 点击“运行报告”Tab。
3. 展开“查看 Markdown 报告”。

### 查看历史记录

1. 运行完成后查看页面底部“历史运行记录”。
2. 选择 run_id。
3. 点击“加载为当前结果”或展开历史代码、错误日志、pytest 测试和质量评分。

## 交互边界

- 不在 Web UI 中实现复杂拖拽、弹窗系统或前后端分离。
- 不改变 LangGraph 节点逻辑。
- 不让演示模式隐藏关键失败信息。
- 不让开发模式丢失完整调试信息。
