# v1.0 版本冻结规则

从现在开始，项目进入 v1.0 比赛演示版冻结阶段。目标是保护演示稳定性，避免临近提交前引入大范围风险。

## 允许修改

v1.0 冻结后只允许以下类型修改：

- 修复 Bug。
- 调整文案。
- 优化文档。
- 修复启动脚本。
- 修复 Web UI 小问题。
- 修复依赖安装说明。
- 修复 Docker 启动配置问题。
- 修复明显影响比赛演示的稳定性问题。

## 禁止修改

v1.0 冻结后禁止以下类型修改：

- 新增大功能。
- 大改 LangGraph 流程。
- 大改插件系统。
- 新增 Vue / Java / C++ 实际代码。
- 改变核心执行逻辑。
- 重构 Agent 调用方式。
- 改变报告和历史记录核心结构。
- 引入数据库或复杂前后端分离。
- 大范围调整 Web UI 交互。

## 代码修改原则

- 优先小改，避免连带影响。
- 修改前确认不影响 CLI、Web UI、Docker 和离线模式。
- 修改后至少运行语法检查或启动冒烟测试。
- 不提交 `.env`、API Key、自动生成报告、自动生成历史记录和输出文件。
- 不覆盖用户本地配置。

## 文档修改原则

- README 保持比赛提交版简洁清晰。
- 操作手册和演示流程优先保证可照着执行。
- 答辩材料优先保证表达准确、简洁、能解释项目价值。
- v2.0 内容只作为规划，不加入实际多语言工程代码。

## 版本边界

v1.0 的目标是：

- 稳定演示。
- 清晰答辩。
- 可部署。
- 可复盘。

v2.0 的目标是：

- 前后端分离。
- 多语言平台化。
- 更强代码沙箱。
- 更完整工程交付。

二者边界必须保持清晰，v1.0 不再引入 v2.0 的实际工程模块。
